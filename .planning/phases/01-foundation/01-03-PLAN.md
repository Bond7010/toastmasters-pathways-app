<!-- W3C APG ARIA tab pattern requirements (implement verbatim — from PITFALLS.md M-5) -->

Required attributes on container: role="tablist"
Required attributes on each tab:
  - role="tab"
  - aria-selected="true" (active) or aria-selected="false" (inactive)
  - tabindex="0" (active) or tabindex="-1" (inactive)
  - aria-controls="content-pane"   ← the panel ID
  - id="tab-{pathId}-l{levelIndex}"

Required attributes on the panel:
  - role="tabpanel"
  - id="content-pane"
  - aria-labelledby="tab-{pathId}-l{activeLevel}"
  - tabindex="0"

Keyboard behavior (W3C APG):
  - ArrowRight: focus moves to next tab (wraps from last to first)
  - ArrowLeft: focus moves to previous tab (wraps from first to last)
  - Home: focus moves to first tab
  - End: focus moves to last tab
  - Enter / Space: activates focused tab (calls setState with the focused tab's level index)
  - Tab key: moves focus out of the tablist to the panel or next focusable element

Note: Focus movement (ArrowLeft/Right) does NOT automatically activate the tab in APG's
"manual activation" pattern. Enter/Space is required. This is the correct pattern here
because auto-activation on arrow keys would trigger a render on every arrow press.

    **1. Keyboard handler on #tab-strip:**
    This implements the W3C APG "manual activation" keyboard pattern. Arrow keys move focus between tabs; Enter/Space activates the focused tab.

    ```javascript
    document.getElementById('tab-strip').addEventListener('keydown', (e) => {
      const tabs = Array.from(document.querySelectorAll('#tab-strip [role="tab"]'));
      const currentIndex = tabs.indexOf(document.activeElement);
      if (currentIndex === -1) return; // focus not on a tab

      let newIndex = currentIndex;

      switch (e.key) {
        case 'ArrowRight':
          newIndex = (currentIndex + 1) % tabs.length;
          e.preventDefault();
          break;
        case 'ArrowLeft':
          newIndex = (currentIndex - 1 + tabs.length) % tabs.length;
          e.preventDefault();
          break;
        case 'Home':
          newIndex = 0;
          e.preventDefault();
          break;
        case 'End':
          newIndex = tabs.length - 1;
          e.preventDefault();
          break;
        case 'Enter':
        case ' ':
          // Activate the currently focused tab
          const levelIndex = parseInt(tabs[currentIndex].dataset.levelIndex, 10);
          setState({ selectedLevel: levelIndex });
          e.preventDefault();
          return; // focus stays on current tab; no focus movement
        default:
          return; // let other keys propagate normally
      }

      // Move focus to new tab (does NOT activate it — manual activation pattern)
      tabs[newIndex].focus();
    });
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User gesture (click/keydown) → event handler | User-controlled input enters event system |
| Event handler → setState() → render() | Unvalidated values from DOM events flow into state |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | Tab click delegation — dataset.levelIndex | mitigate | Parse with `parseInt(tab.dataset.levelIndex, 10)`. Always bounds-check before array access: `if (levelIndex < 0 || levelIndex >= pathData.levels.length) return;` inside setState/render. Malformed data-level-index on injected DOM elements cannot cause script errors. |
| T-03-02 | Tampering | Path selector value — e.target.value | mitigate | Before calling setState, validate: `if (!PATHWAYS.find(p => p.id === e.target.value)) return;`. This ensures only valid path slugs reach the state. The `<select>` options are generated from PATHWAYS itself, so tampering requires console access (accepted risk for a public reference tool). |
| T-03-03 | Denial of Service | IntersectionObserver on scroll sentinel | accept | Observer is disconnected implicitly when elements are removed from DOM. No runaway callbacks possible. Single observer instance. |
| T-03-04 | Elevation of Privilege | Keyboard event propagation | accept | `e.preventDefault()` on handled keys prevents browser default behaviors (e.g., page scroll on Space). No privilege escalation possible in a static HTML file. |
</threat_model>
