<objective>
Add a `validatePathwaysData()` function to `pathways-app.html` that can be called from the browser DevTools console to validate the embedded PATHWAYS JSON data. The function must produce a clear PASS or FAIL report listing any violations found.
</objective>

<requirements>
The function must check all five categories:

1. **Time format** — every `project.time` must match the regex `/^\d+–\d+ min$/` (en-dash U+2013, not hyphen; abbreviated unit; no "minute" word)
2. **Type enum** — every `project.type` must be exactly `"Required"` or `"Elective"` (case-sensitive; no other values)
3. **Smart quotes / curly apostrophes** — no `"`, `"`, `'`, `'` characters in any string field: `project.name`, `project.time`, `project.purpose`, `level.name`, `level.completionRequirement`, `path.name`
4. **Enhancement field** — for vintage paths (`"basic-training"` and `"communication-series"`), every level's `enhancement` must be `null`; for all 6 current paths, every level's `enhancement` must be non-null
5. **Duplicate project names** — within a single path+level combination, no two projects may share the same `name`

Output format: `console.log` a PASS or FAIL banner, then list each violation with path/level/project context.
</requirements>

<tasks>
1. Add the `validatePathwaysData()` function to the `<script>` block in `pathways-app.html`, placed just after the `PATHWAYS` constant is defined (after `var PATHWAYS = _pd.paths;`).
2. The function must be globally accessible from the console — assign it to `window.validatePathwaysData`.
3. Commit: `feat(02-01): add validatePathwaysData() console validator`
</tasks>

<must_haves>
- Function is callable as `validatePathwaysData()` in browser console
- Reports exact path name, level number, and project name for each violation
- Clearly distinguishes PASS from FAIL in output
- Does not modify any runtime behaviour — pure read + console.log
</must_haves>
